## Contexto

O David Store foi pensado como um ecossistema de e-commerce moderno, não apenas como uma vitrine estática.
O projeto junta navegação pública, fluxo de compra e base administrativa em uma mesma narrativa técnica.

## O que o produto precisava entregar

- experiência de catálogo com leitura rápida e boa organização
- carrinho e checkout como fluxo contínuo, não como páginas desconexas
- backend com autenticação, persistência e rotas coerentes
- base preparada para operação e observabilidade

## Decisões de implementação

### Frontend orientado a experiência de produto

O uso de `Next.js` atende bem um e-commerce porque permite equilibrar performance, renderização híbrida e estrutura de navegação clara.
Esse tipo de produto depende de velocidade percebida, hierarquia visual e previsibilidade no fluxo.

### Backend separado por responsabilidade

A API em `Node.js + Express + Prisma` concentra regras de dados, autenticação e endpoints do domínio.
Isso evita empurrar responsabilidade de negócio para o frontend e facilita manutenção da aplicação.

### Infra preparada para evolução

O projeto considera execução com Docker, camadas compartilhadas e estrutura suficiente para crescer em complexidade sem exigir reescrita completa.

## Por que esse case entra no portfólio

- mostra equilíbrio entre interface e backend
- demonstra repertório em produto de alto atrito comercial
- reforça minha capacidade de organizar um fluxo completo, do catálogo à base operacional

## O que esse projeto revela sobre meu perfil

Mesmo em um contexto mais visual, a minha abordagem continua a mesma: estruturar o sistema para que a experiência do usuário seja consequência de uma base técnica coerente.
